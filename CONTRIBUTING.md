# Contributing to blaue-tonne-api

## Commit Message Format

**Use conventional commit prefixes:**

```
<type>: <short description>
```

### Types

- `feat:` - New feature or functionality
- `fix:` - Bug fix
- `docs:` - Documentation changes
- `style:` - Code style/formatting (no logic change)
- `refactor:` - Code restructuring (no behavior change)
- `test:` - Adding or updating tests
- `chore:` - Maintenance (deps, config, build)

### Guidelines

- **Keep it short and concise** (max 50 chars for subject)
- **No period at the end**
- **Imperative mood** ("add feature" not "added feature")
- **Lowercase after prefix**

### Examples

```
feat: add Wasserburg district to plans.yaml
fix: handle PDFs with missing tables
docs: update README with Docker instructions
test: add parametrized tests for all districts
chore: bump Python to 3.14
refactor: extract date parsing to separate function
```
